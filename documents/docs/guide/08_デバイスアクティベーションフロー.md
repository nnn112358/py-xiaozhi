# デバイスアクティベーションフロー v2

## 概要

現在のフローはシャコデバイス認証 v2 バージョンです

## アクティベーションフロー

各デバイスには一意のシリアルナンバー (Serial Number) と HMAC キー (HMAC Key) があり、身元確認とセキュア通信に使用されます。新しいデバイスを初めて使用する場合、以下のフローでアクティベーションを行う必要があります：

1. クライアント起動時、サーバーにデバイス情報を送信（シリアルナンバー、MAC アドレス、クライアント ID を含む）
2. サーバーがデバイスがアクティベート済みかどうかをチェック：
   - アクティベート済みの場合、クライアントは通常動作
   - 未アクティベートの場合、サーバーは認証コードと Challenge を含むアクティベーションリクエストを返す
3. クライアントが認証コードを表示し、ユーザーに xiaozhi.me ウェブサイトで認証コードを入力するよう促す
4. クライアントが HMAC キーで Challenge に署名し、サーバーに送信して検証
5. クライアントがポーリング方式でサーバーの検証結果を待機：
   - 検証成功の場合、デバイスアクティベーション完了
   - 検証失敗またはタイムアウトの場合、デバイスアクティベーション失敗

### 3. 设备激活请求 (POST /xiaozhi/ota/activate)

**请求体**:
```json
{
  "Payload": {
    "algorithm": "hmac-sha256",
    "serial_number": "SN-5CD8467B47FB4920",
    "challenge": "dac852d6-4ac4-4650-ba1a-c2a5bf00a766",
    "hmac": "ada4775e3ed93cf9c0eb9ed00444138554ba416af41283a0e5603c77681a8022"
  }
}
```

### 4. 激活响应

- **成功**: 状态码 200
- **等待用户输入验证码**: 状态码 202
- **失败**: 状态码 4xx (如401表示未授权，400表示请求错误)

**响应体** (失败时):

```json
{
  "error": "错误原因描述"
}
```


## 安全机制

设备激活流程v2版本采用以下安全机制：

1. **设备唯一标识**：每个设备有一个唯一的序列号(Serial Number)
2. **HMAC签名验证**：使用HMAC-SHA256算法对Challenge进行签名，确保设备身份的真实性
3. **验证码验证**：通过要求用户在网页端输入验证码，防止自动化的激活攻击
4. **轮询等待机制**：使用HTTP Long Polling等待服务器验证结果，适应各种网络环境
